## Windows 패킷 캡처 에이전트

### 기술 스택
- **언어**: Python 3.11+ (Scapy) 또는 C# (.NET, SharpPcap)
- **패킷 캡처**: Npcap (WinPcap 후속)
- **전송**: WebSocket (ws/wss)

### 핵심 기능
1. **Aion2.exe 프로세스 감지** (Win32 API)
2. **네트워크 인터페이스 자동 선택** (물리/가상/루프백)
3. **BPF 필터**: `tcp and port 13328`
4. **TLS 필터링**: TLS 패킷 무시 (content_type 0x14-0x17)
5. **매직 바이트 프레임 분리**: `06 00 36`
6. **WebSocket으로 Mac 서버에 실시간 전송**

### 트래픽 자동 감지 (taengu 방식 채택)
```python
# 의사코드
1. 모든 NIC에서 TCP 캡처 시작
2. 가상 디바이스(VPN/루프백) 우선
3. 각 TCP 페이로드에서:
   a. TLS 패킷이면 무시
   b. 06 00 36 매직 바이트 검색
   c. 매직 발견 → 후보 포트 등록
   d. 파싱 성공 → 포트/디바이스 잠금
4. 잠금된 트래픽만 서버로 전송
```

## Mac 서버 패킷 파서

### 기술 스택
- **언어**: TypeScript (Node.js)
- **프레임워크**: Fastify + ws
- **DB**: PostgreSQL + Redis

### 파싱 파이프라인
```
Raw bytes → VarInt 길이 파싱 → LZ4 압축 해제 (해당시)
  → Opcode 추출 → 라우터 분기
    → 거래소 패킷: 아이템/가격 파싱 → DB 저장
    → 기타 패킷: 로깅/무시
```

### 거래소 패킷 식별 전략
Aion 1의 `S_VENDOR` 패턴을 참고하여:
1. 거래소 UI 열기/닫기 이벤트 패킷 식별
2. 아이템 목록 응답 패킷 식별 (대량 아이템 데이터 포함)
3. 가격 정보 패킷 식별
4. Opcode 후보를 수동 캡처로 좁혀감

### 리버싱 프로세스
```
Phase 1: 패킷 덤프
  - 거래소 열기 전/후 패킷 비교
  - 검색 실행 전/후 패킷 비교
  - 페이지 넘기기 전/후 패킷 비교

Phase 2: Opcode 식별
  - 반복 패턴의 opcode 추출
  - 알려진 opcode (04 38 등) 제외
  - 미지 opcode 중 거래소 타이밍 매칭

Phase 3: 구조 파싱
  - VarInt 필드 경계 추정
  - 아이템 ID, 가격, 수량 패턴 매칭
  - Aion 1 구조와 비교 검증
```
